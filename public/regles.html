Règles

  1. Les équipes peuvent être composées d’une à six personnes. Le degré de difficulté additionnel pour une équipe plus petite sera pris en considération dans le jugement final.
  2. La planification du projet pendant le 5 à 7 est permise. Des idées et des maquettes peuvent être discutées, mais le code ne doit s’écrire que pendant l’événement le 1er décembre.
  3. L’application doit être au minimum un prototype fonctionnel pour pouvoir se qualifier pour un prix.
  4. L’application doit utiliser au minimum une des API des partenaires et commanditaires afin de se qualifier pour un prix.
  5. Les représentants d’une API peuvent désigner une personne sur place pour juger en leur nom.
  6. Les juges apprécieront les applications selon leur point de vue et selon les recommandations de l’API qu’ils représentent, ainsi que selon la grille de critères fournie par les organisateurs.
